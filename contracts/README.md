# Aether Dark Pool — Soroban Smart Contracts

Five Soroban contracts implementing a zero-knowledge dark pool DEX on Stellar.
Orders are sealed with Groth16 proofs, matched via 60-second batch auctions,
and settled atomically on-chain.

## Architecture

```
ZKVerifier ──────────────────────────────────────────────────────┐
  verify_order_proof / verify_balance_proof /                     │
  verify_range_proof / verify_match_proof                         │
                                                                  ▼
OrderBook ──────► MatchingEngine ──────► Settlement ──────► EscrowVault
  submit_order       submit_match          settle              deposit
  (ZK-gated)         (match proof)                            release
```

### Contracts

| Contract | Description |
|---|---|
| **zk_verifier** | Real BN254 Groth16 verification via Stellar host functions. Holds VKs for order, balance, range, and match circuits. |
| **escrow_vault** | Non-custodial vault. Locks trader funds on deposit; releases `amount` to counterparty and refunds `deposit − amount` to trader atomically. |
| **order_book** | Accepts sealed orders gated by three ZK proofs (order commitment, balance, range). Stores commitment + nullifier on-chain. |
| **settlement** | Executes a matched pair: calls EscrowVault.release for buyer and seller, emits settlement event. |
| **matching_engine** | Relayer-driven batch processor. Verifies a Groth16 match proof before calling Settlement.settle, ensuring no forged clearing price. |

## Build

Requires Rust nightly with the `wasm32v1-none` target (Soroban SDK 26.x):

```bash
rustup target add wasm32v1-none
make build        # builds all 5 contracts in dependency order
make optimize     # runs stellar contract optimize on each WASM
```

Or build a single package:

```bash
cargo build --release --target wasm32v1-none --package zk_verifier
```

## Test

```bash
make test         # cargo test — 7 ZKVerifier unit tests + escrow/order/settlement tests
```

Test coverage includes:
- Real Groth16 proof verification (BN254 test vectors generated by snarkjs)
- Tampered signal rejection
- Wrong proof rejection
- Signal count mismatch rejection
- Match proof verification and tampered clearing price rejection
- Double-initialize guard
- EscrowVault deposit/release/refund arithmetic

## Deploy

### Prerequisites

```bash
cargo install --locked stellar-cli --features opt
```

Set environment variables:

| Variable | Description |
|---|---|
| `STELLAR_SECRET_KEY` | Deployer/admin secret key (S...) |
| `STELLAR_NETWORK_PASSPHRASE` | `Test SDF Network ; September 2015` for testnet |

### Automated deploy

```bash
export STELLAR_SECRET_KEY=SXXX...
make deploy       # deploys all optimized WASMs to testnet
```

### Manual deploy (recommended)

```bash
cd contracts

# 1. Build + optimize
make build && make optimize

# 2. Deploy each contract
stellar contract deploy \
  --wasm target/wasm32v1-none/release/zk_verifier.optimized.wasm \
  --source "$STELLAR_SECRET_KEY" \
  --network testnet

# 3. Initialize (see scripts/initialize.sh)
bash scripts/initialize.sh
```
